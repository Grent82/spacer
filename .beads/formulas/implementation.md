# Spacer Implementation Template

Use this template for all implementation tasks (features, rules, services, entities).

## Required Sections

### Context
Why does this work exist? What problem does it solve? What source material (rai7 labels, variable mappings, design docs) informs this implementation?

### Goals
What should be achieved? List concrete, testable outcomes.

### Scope
**In Scope:**
- Specific files, classes, or behaviors to implement/change

**Out of Scope:**
- Related work that should be handled separately

### Implementation Plan
1. Step one
2. Step two
3. Step three

### Acceptance Criteria
- [ ] Criterion 1 (testable)
- [ ] Criterion 2 (testable)
- [ ] Criterion 3 (testable)

### Design Decisions
Any architectural choices, trade-offs, or patterns used. Reference existing patterns where applicable.

### Notes
Additional context, open questions, or references to related issues.

---

## Usage

Create issues with:
```bash
bd create --title="Implement X" --description="<paste template>" --type=task --priority=2
```

Fill in all sections before starting work.
